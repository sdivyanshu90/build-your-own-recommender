# Two-Tower Recommender documentation

This repository follows recommendation artifacts from exposure events through a retrieval API.
Start with [recommendation concepts](recommendation-concepts.md), then read
[two-tower theory](two-tower-theory.md) and [architecture](architecture.md). The quick-start and
all executable commands are maintained in the repository README.

The default pipeline is intentionally local and deterministic. Production extensions—managed
feature stores, distributed preprocessing, object storage, a richer ranker, stream processing,
and online experimentation—are called out as extensions rather than claimed implementations.
